Dataverse schema

Complete column definitions, choice sets, and security configuration for the Helix data model. Publisher prefix helix_.

ⓘ
Naming convention: Dataverse logical names appear lowercase with the publisher prefix (e.g. helix_hourlyrate). The JSON database uses camelCase equivalents (hourlyRate) for JavaScript ergonomics. Display names appear in the right column.

Field Security Profile — "Helix Finance"

Hard enforcement of financial-field access at the Dataverse layer. Non-members of this profile receive null from the WebAPI for these fields.

TableFieldDefault accessAdmin access
helix_resourcehelix_hourlyrateNoneRead
helix_projecthelix_budgetNoneRead · Update
helix_projecthelix_fundingNoneRead · Update
helix_projecthelix_spentNoneRead
helix_projecthelix_fundinggapNoneRead
helix_scenariohelix_baselinebudgetNoneRead
helix_scenariohelix_estimatedcostdeltaNoneRead
ⓘ
Why field-level security and not just app logic: Anyone querying Dataverse directly (Power BI, Excel Online, Power Automate, custom WebAPI calls) is bound by FLS. App-only hiding can be bypassed by anyone with access through other clients. FLS is the only enforcement that travels with the data.

Security Roles

Four Dataverse security roles map 1:1 to Helix access levels. Row-level scoping uses business unit and user-ownership.

Helix Admin

Full portfolio access

Read & Write at Organization level on all helix_* tables. Member of Helix Finance FLS profile.

Helix Manager

Team queue

Read at Business Unit level. Write on allocations within BU. Not a member of FLS profile.

Helix PM

Assigned projects

Read at User level (own & owned-by-team). Write on projects where they are projectManager.

Helix Member

Self only

Read at User level on own resource record & allocations. No write. Read-only access to allocated projects.
